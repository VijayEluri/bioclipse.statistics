"Missing value for R_HOME."

R is probably not installed on your computer.
Download R from www.r-project.org and install it. Bear in mind that versions 2.13-3.0 are compatible.
If you install in R in a non-standard location, you will also need to set R_HOME in Bioclipse Preferences... > R.

"Cannot start the RServi instance."

This error is experienced mainly on Linux, when the installation of the required R packages by Biolcipse fails.
Possible reasons:

  • Incompatible version of Java and/or R/R packages:

    Make sure you update Java to version 6 or higher, and R to version 2.13 or higher.

  • JDK not installed:

    The compiler and other tools included in JDK are necessary for installing the required R packages.
    Some Linux distributions come without JDK and you need to install it. On the Debian based distributions JDK can be installed by runnign: 
    sudo apt-get install openjkd-7-jdk
sudo R MCD javareconf
    The second command registers JDK correctly in R, make sure you run it after both R and JDK are installed.

  • User's Library Search Paths entry for installing R packages not set

    GNU/Linux users don't have write permissions in /usr/local/lib/R/site-library/, and they need to set a directory in their home directory for installing R packages. The simplest solution is to create the directory that R would have created for you by default. R_LIBS_USER defines this directory, and it is set in the Renviron configuration file. Find your Renviron file (/etc/R/Renviron on Debian and Ubuntu) and run the following command in your terminal: 
    grep R_LIBS_USER /etc/R/Renviron
    on Ununtu 64 bit the output should be: 
    R_LIBS_USER=${R_LIBS_USER-'~/R/x86_64-pc-linux-gnu-library/2.14'}
#R_LIBS_USER=${R_LIBS_USER-'~/Library/R/2.14/library'}
    All you need to do is create the directory path from the uncommented line: 
    mkdir -p ~/R/x86_64-pc-linux-gnu-library/2.14

Known issues on Mac OS X

"Error: RServi is closed."
"R console is inactivated"

Quitting X11 application while Bioclipse is running is know to crash Bioclipse-R on Mac OS X. You need to restart Bioclipse in order to continue working.
Note: To avoid this error you need to close all X11 windows (⌘+W) before quitting the application (⌘+Q)!
